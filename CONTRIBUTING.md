## 🚀 Как начать

### 1. Форк репозитория

1. Перейдите на страницу репозитория
2. Нажмите кнопку "Fork" в правом верхнем углу
3. Клонируйте ваш форк:

```bash
git clone https://github.com/YOUR_USERNAME/HeartCare-AI.git
cd HeartCare-AI
```

### 2. Настройка окружения

```bash
# Создание виртуального окружения
python -m venv venv

# Активация (Windows)
venv\Scripts\activate

# Активация (Linux/macOS)
source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 3. Создание ветки

```bash
git checkout -b feature/your-feature-name
```

## 🎨 Стиль кода

### Общие правила

1. **Язык**: Все комментарии и документация на русском языке
2. **PEP 8**: Соблюдайте стандарт PEP 8
3. **Type hints**: Используйте type hints для функций
4. **Docstrings**: Добавляйте docstrings для всех функций и классов

### Структура файлов

```python
# 1. Импорты (в алфавитном порядке)
import os
from typing import Optional

import pandas as pd
import numpy as np

# 2. Константы (в верхнем регистре)
CONSTANT_VALUE = 42

# 3. Классы
class MyClass:
    """Класс для ..."""
    
    # 4. Метод __init__
    def __init__(self, param: str):
        self.param = param
    
    # 5. Публичные методы
    def public_method(self):
        """Публичный метод"""
        pass
    
    # 6. Приватные методы
    def _private_method(self):
        """Приватный метод"""
        pass
    
    # 7. Статические методы
    @staticmethod
    def static_method():
        """Статический метод"""
        pass
    
    # 8. Свойства
    @property
    def property_name(self):
        """Свойство"""
        return self._value
```

## 🔄 Процесс внесения изменений

### 1. Создание ветки

```bash
git checkout -b feature/your-feature-name
```

### 2. Внесение изменений

- Создавайте маленькие, сфокусированные коммиты
- Один коммит = одна логическая единица изменений
- Пишите понятные сообщения коммитов

### 3. Сообщения коммитов

Формат: `тип: краткое описание`

Примеры:
```
feat: добавление модели CatBoost
fix: исправление ошибки в предобработке данных
docs: обновление README
refactor: рефакторинг data_loader.py
test: добавление unit-тестов
```

### 4. Локальное тестирование

```bash
# Запуск основного скрипта
python scripts/train.py

# Проверка структуры
python -c "from src.data_loader import DataLoader; print('OK')"
```

### 5. Push в ваш форк

```bash
git add .
git commit -m "feat: ваше описание"
git push origin feature/your-feature-name
```

### 6. Создание Pull Request

1. Перейдите на GitHub
2. Нажмите "Compare & pull request"
3. Заполните шаблон PR:
   - **Заголовок**: Краткое описание
   - **Описание**: 
     - Что изменено
     - Почему изменено
     - Как тестировалось
   - **Связанные issues**: Укажите номера issues

## 🧪 Тестирование

### Ручное тестирование

Перед созданием PR убедитесь, что:

```bash
# 1. Установка зависимостей
pip install -r requirements.txt

# 2. Запуск основного скрипта
python scripts/train.py

# 3. Проверка результатов
ls results/models/
ls results/plots/

# 4. Проверка Jupyter
jupyter notebook notebooks/analysis.ipynb
```

### Автоматическое тестирование (будет добавлено)

```bash
# Запуск тестов
python -m pytest tests/

# Проверка покрытия
python -m pytest --cov=src tests/
```

## 📚 Документация

### Обновление README

Если вы добавляете новую функциональность:

1. Обновите раздел "Возможности системы"
2. Добавьте примеры использования
3. Обновите структуру проекта
4. Добавьте метрики, если применимо

### Добавление docstrings

Все новые функции и классы должны иметь docstrings:

```python
def new_function(param: str) -> int:
    """
    Краткое описание функции
    
    Подробное описание того, что делает функция,
    какие параметры принимает и что возвращает.
    
    Args:
        param: Описание параметра
        
    Returns:
        int: Описание возвращаемого значения
        
    Raises:
        ValueError: Если параметр некорректен
    """
    pass
```

## 📊 Качество кода

### Проверки перед PR

- [ ] Код соответствует PEP 8
- [ ] Все комментарии на русском языке
- [ ] Добавлены docstrings
- [ ] Используются type hints
- [ ] Код работает без ошибок
- [ ] Документация обновлена
- [ ] Тесты проходят (если есть)

### Инструменты для проверки

```bash
# Проверка стиля кода
pip install flake8
flake8 src/

# Проверка типов
pip install mypy
mypy src/

# Форматирование
pip install black
black src/
```

## 🎯 Типы вносимых изменений

### 1. Новые функции (feat)

- Добавление новых моделей
- Новые методы визуализации
- Улучшение предобработки данных

### 2. Исправления (fix)

- Исправление ошибок
- Улучшение производительности
- Оптимизация памяти

### 3. Документация (docs)

- Обновление README
- Добавление примеров
- Улучшение комментариев

### 4. Рефакторинг (refactor)

- Улучшение структуры кода
- Оптимизация логики
- Удаление дублирования

### 5. Тесты (test)

- Добавление unit-тестов
- Интеграционные тесты
- Тесты на качество данных

## 🚫 Что не принимается

- Код без комментариев на русском языке
- Код без docstrings
- Код без type hints
- Код, который не работает
- Код без тестирования
- Код, нарушающий архитектуру проекта

## 📞 Поддержка

Если у вас есть вопросы:

1. Изучите документацию в первую очередь
2. Проверьте существующие issues
3. Создайте новый issue с вопросом
4. Опишите проблему подробно

## 🎓 Пример хорошего вклада

### Хороший PR

```bash
# Ветка
git checkout -b feature/add-catboost-model

# Изменения
- Добавлена модель CatBoost в src/models/catboost_model.py
- Обновлен config/settings.py
- Добавлены примеры использования в README.md
- Протестировано на тестовых данных

# Коммиты
git commit -m "feat: добавление модели CatBoost"
git commit -m "docs: обновление README"
```

### Плохой PR

```bash
# Ветка
git checkout -b my-changes

# Изменения
- Много несвязанных изменений
- Нет комментариев
- Нет тестов
- Нет документации
```
